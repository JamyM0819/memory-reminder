# 艾宾浩斯记忆提醒器 — 设计规格

## 概述

一个极简的 PWA（渐进式 Web 应用），用户记录任意备忘录（单词、代码片段等），系统依据**艾宾浩斯遗忘曲线**的 11 个复习间隔自动在 Android 手机上推送提醒通知。支持自定义免打扰时段（工作日/休息日可分别设置），落在免打扰期的提醒自动顺延。无复杂交互，到点通知，看一眼即完成回忆。

## 用户流程

1. 用户用 Chrome 打开 PWA 链接 → 添加到主屏幕 → 允许通知权限
2. 点击 "+" → 输入纯文本 memo（如 `JS 数组去重 → [...new Set(arr)]`）→ 保存
3. 服务器自动生成 11 条提醒计划，按以下间隔推送通知：

   | 序号 | 间隔 |
   |------|------|
   | 1 | 5 分钟 |
   | 2 | 30 分钟 |
   | 3 | 1 小时 |
   | 4 | 3 小时 |
   | 5 | 6 小时 |
   | 6 | 12 小时 |
   | 7 | 1 天 |
   | 8 | 2 天 |
   | 9 | 4 天 |
   | 10 | 7 天 |
   | 11 | 15 天 |

4. 若提醒时间落在免打扰时段内，自动顺延至免打扰结束后立即发送
5. 通知直接显示 memo 全文，用户看到即完成回忆
6. 首页列表显示所有 memo、当前进度（已提醒 N/11）、下次提醒时间

## 技术架构

```
┌─────────────┐     ┌──────────────┐     ┌──────────┐
│  Chrome PWA  │────▶│  Node.js API  │────▶│  SQLite  │
│ (前端静态页)  │◀────│  (Express)    │◀────│  (数据)   │
└─────────────┘     └──────┬───────┘     └──────────┘
                           │
                    ┌──────▼───────┐
                    │  Web Push    │
                    │  (通知推送)   │
                    └──────────────┘
```

### 前端（PWA）
- 纯静态 HTML/CSS/JS，无框架
- manifest.json + Service Worker 支持离线缓存 + 添加到主屏幕
- 三个主视图：memo 列表、添加页、设置页
- 支持 Web Push 订阅（用户授权后生成端点 → 发给后端）

### 后端（Node.js + Express）
- RESTful API：CRUD memo + 管理推送订阅
- 定时任务（node-cron）：每分钟检查到期提醒，批量推送
- SQLite 数据库，无需额外安装

### 推送机制
- 使用 Web Push Protocol + VAPID keys
- Android Chrome 全后台支持通知接收
- 免费推送，无需第三方推送服务

## 数据库设计

### memos 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| content | TEXT | memo 内容 |
| created_at | DATETIME | 创建时间 |
| is_active | BOOLEAN | 是否启用（可暂停） |

### reminders 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| memo_id | INTEGER FK | 关联 memo |
| interval_index | INTEGER | 第几次 (0-10) |
| remind_at | DATETIME | 计划提醒时间 |
| notified | BOOLEAN | 是否已推送 |

### subscriptions 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| endpoint | TEXT | 推送端点 |
| keys | TEXT | 推送密钥(JSON) |

### settings 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| key | TEXT UNIQUE | 设置键名 |
| value | TEXT | 设置值(JSON) |

存储的设置项：
- `dnd_weekday_start` / `dnd_weekday_end` — 工作日免打扰起止时间（如 "23:00" / "07:00"）
- `dnd_weekend_start` / `dnd_weekend_end` — 休息日免打扰起止时间（如 "23:00" / "09:00"）

## API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/memos | 获取所有 memo 及进度 |
| POST | /api/memos | 创建 memo，自动生成 11 条提醒 |
| PUT | /api/memos/:id | 更新 memo 内容 |
| DELETE | /api/memos/:id | 删除 memo |
| PUT | /api/memos/:id/toggle | 启用/暂停提醒 |
| POST | /api/subscribe | 注册推送订阅 |
| POST | /api/unsubscribe | 取消订阅 |
| GET | /api/settings | 获取所有设置 |
| PUT | /api/settings | 更新设置 |

## 免打扰逻辑

1. 定时任务每分钟扫描待发送的提醒
2. 对于 `remind_at <= now` 且 `notified = 0` 的提醒：
   - 判断当天是工作日（周一~周五）还是休息日（周六、周日）
   - 取对应的免打扰时间段
   - 若当前时间落在免打扰段内 → **跳过本次扫描**（下次再检查）
   - 若当前时间不在免打扰段内 → **立即推送**，标记 notified
3. 若提醒时间已过去但不在免打扰段内，立即发送（不丢提醒）

## 界面设计

极简，移动端优先（375px 基准）：

### 首页（memo 列表）
- 顶栏标题 "记忆提醒" + 设置 ⚙ 图标
- 卡片列表，每个卡片显示：
  - memo 内容（单行截断）
  - 进度点：● ● ● ● ● ● ● ● ○ ○ ○（实心=已提醒）
  - 下次提醒时间
- 右下角 FAB 浮动 "+" 按钮

### 添加页
- 大文本框（自动聚焦，回车换行）
- 保存按钮
- 取消返回

### 设置页
- 工作日免打扰：开始时间选择器 / 结束时间选择器
- 休息日免打扰：开始时间选择器 / 结束时间选择器

## 部署

部署到 Railway 或 Fly.io 免费层：
- Node.js 进程保持运行，定时任务即可持续工作
- SQLite 用文件存储，内置备份机制
- 提供 HTTPS URL

## 非功能性需求

- 首次加载 < 2s（纯静态，无框架）
- 通知延迟 < 60s（cron 每分钟扫一次）
- 单用户设计，无登录系统
